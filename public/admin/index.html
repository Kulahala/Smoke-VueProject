📖 B2B Catalogue 操作指南

📌 核心注意事项

🚨 新增商品 ID 限制
每次新增商品时，Product ID（数字 ID）必须是递增的且不可重复！例如，若已有最大 ID 是 13，新商品 ID 必须填写 14。如果填了已有 ID，会直接覆盖该 ID 对应的旧产品数据，导致旧产品永久丢失！前台网站会检测重复 ID 并显示红色警告横幅。
⚠️ 分类 ID 禁止修改
编辑分类时，Group ID 和 Category ID 已经被锁定/隐藏，千万不要尝试更改或随意替换其定义，否则前台网页将无法正确过滤显示商品。

⚙️ 栏目如何管理 (点击看步骤)

🎨 内容与图片规范

  • 商品图片：必须上传纯白色背景的渲染图或实拍图。前台系统会对白底图片自动渲染为高端的“照片展示盒”物理效果。
  • 系列背景图 (Hero Image)：必须是无文字、无规格图表的纯净、高清大特写图，防止在 object-fit cover 或暗色遮罩时文字被裁剪、显示杂乱。

🚀 如何发布上线

  • 所有的修改和新增，都需要点击右上角的 Publish (发布) 按钮。
  • 点击发布后，Netlify 会在后台自动完成打包部署，通常 1-2 分钟内前台网站即可更新。

🔄 下架、删除与回滚

  • 暂时下架商品：在商品编辑页右上角，将状态从 Published 改为 Draft (草稿) 后发布。Draft 状态的商品不会出现在前台目录页和详情页。
  • 永久删除商品：在商品列表页，点击商品条目右侧的 🗑 删除图标，确认后发布即可。删除后前台将无法访问该商品。
  • 数据回滚：所有修改都通过 Git 提交记录保存。如需回滚到之前的版本，请联系技术人员在 GitHub 上操作，或通过 Netlify 控制面板的 Deploys → Rollback 功能回退整个站点到之前的部署版本。
